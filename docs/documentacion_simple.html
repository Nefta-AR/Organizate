1. Arquitectura General del Sistema

Simple es una aplicación móvil multiplataforma (Android/iOS) desarrollada con Flutter. Su arquitectura combina un frontend reactivo con backend serverless de Firebase.

Frontend — Flutter

  • Flutter 3.3+ / Dart 3.3+
  • Material Design 3
  • Provider (PomodoroService)
  • StreamBuilder reactivo
  • flutter_tts (Text-to-Speech)
  • fl_chart (gráficos)
  • flutter_svg (pictogramas ARASAAC)

Backend — Firebase

  • Firebase Auth (email + Google)
  • Cloud Firestore (base de datos)
  • Firebase Storage (imágenes)
  • Cloud Functions (proxy Gemini)
  • Firebase Cloud Messaging (push)

Servicios Externos

  • Gemini 2.0/2.5 Flash (IA)
  • Google Sign-In (OAuth)
  • ARASAAC (banco de SVGs)
  • flutter_local_notifications
  • url_launcher (llamadas)

Patrón de State Management

La app usa dos estrategias de estado según el alcance:

EstrategiaCuándo se usaEjemplo
StreamBuilderEstado persistido en Firestore que debe reflejarse en tiempo realTareas, puntos, feature flags del tutor
ChangeNotifier + ProviderEstado en memoria que debe persistir entre pantallas sin FirestorePomodoroService (timer)
setStateEstado local efímero de un widget (loading, formularios)_isLoading en LoginScreen

2. Flujo de Navegación y Rutas

Arquitectura de Navegación en 3 Niveles

La app usa una máquina de estados reactiva implementada en AuthGate en lugar de un router convencional. Cada nivel es un StreamBuilder independiente:

Nivel 1 — AuthGate
Stream: FirebaseAuth.authStateChanges()
• Sin sesión → LoginScreen
• Con sesión → Nivel 2
↓
Nivel 2 — _UserOnboardingGate
Stream: users/{uid} (Firestore)
• role == null o hasCompletedOnboarding == false → RoleSelectionScreen
• hasCompletedProfile == false y name vacío → ProfileSetupScreen
• Todo OK → Nivel 3
↓
Nivel 3 — RoleDispatcher
Switch sobre el campo role del documento Firestore
• role == 'tutor' → TutorSupervisarScreen
• role == 'usuario' (o cualquier otro) → HomeScreen

Rutas Nombradas (AppRouter)

Archivo: lib/core/router/app_router.dart

ConstanteRutaPantalla destinoTransición
AppRouter.onboarding/onboardingOnboardingScreenFade 200ms
AppRouter.home/homeHomeScreenFade 200ms
AppRouter.usuarioTea/usuario-teaPantallaUsuarioTEAFade 200ms
AppRouter.vincularUsuario/vincular-usuarioVinculacionTutorScreenFade 200ms
Las transiciones usan FadeTransition de 200ms en lugar del slide estándar de MaterialPageRoute, para reducir la carga perceptual en usuarios con TEA y TDAH.

Barra de Navegación Inferior (CustomNavBar)

Archivo: lib/core/widgets/custom_nav_bar.dart

La barra de navegación es dinámica: las pestañas disponibles dependen de los feature flags del documento pictogramSettings/_features en Firestore.

3. Módulo de Autenticación

3.1 LoginScreen — Inicio de sesión / Registro

Archivo: lib/features/auth/screens/login_screen.dart

Acceso: Pantalla raíz cuando no hay sesión activa (AuthGate nivel 1)

Propósito

Pantalla dual que alterna entre modo LOGIN y modo REGISTRO usando el booleano _isLogin. Ofrece autenticación con email/contraseña y con Google OAuth.

Funciones Principales

FunciónQué haceDetalle técnico
_loadSavedEmail() Pre-rellena el campo email con el último usado Lee la clave 'saved_email' de SharedPreferences al iniciar el widget
_saveEmail(email) Guarda el email para la próxima sesión Escribe en SharedPreferences después de un login exitoso
_handleLogin() Ejecuta login o registro con email/contraseña LOGIN: signInWithEmailAndPassword
REGISTRO: createUserWithEmailAndPassword + crea documento Firestore users/{uid} con: role='usuario', points=0, streak=0, hasCompletedOnboarding=false
_handleGoogleLogin() Login con cuenta de Google Web: signInWithPopup con prompt 'select_account'
Móvil: GoogleSignIn().signIn() → credenciales OAuth → signInWithCredential
Detecta isNewUser para crear documento Firestore solo en el primer login
_handleForgotPassword() Envía email de recuperación de contraseña Valida que el campo email tenga un '@' antes de llamar sendPasswordResetEmail
_mapAuthError(code) Traduce códigos de error Firebase a mensajes en español Switch sobre: user-not-found, wrong-password, invalid-credential, email-already-in-use, weak-password, invalid-email, too-many-requests, network-request-failed

Validaciones del Formulario

  • Nombre (solo registro): mínimo 2 caracteres
  • Email: debe contener '@'
  • Contraseña: mínimo 6 chars + al menos 1 mayúscula + 1 dígito (regex: ^(?=.*[A-Z])(?=.*\d))

Lógica de UI

  • El campo "Nombre" se muestra/oculta con AnimatedSize al alternar entre login y registro
  • El botón principal muestra CircularProgressIndicator durante la operación async
  • Después de login exitoso: Navigator.pushNamedAndRemoveUntil('/', ...) para limpiar el stack y dejar que AuthGate re-evalúe
  • El botón de Google es un widget separado _GoogleButton con su propio estado de carga

3.2 RoleSelectionScreen — Selección de Rol

Archivo: lib/features/auth/screens/role_selection_screen.dart

Acceso: Cuando role == null o hasCompletedOnboarding == false en Firestore (AuthGate nivel 2)

Propósito

Permite al usuario elegir si usará la app como Usuario (organización personal) o como Tutor (supervisión de otro). Esta elección determina toda la experiencia posterior.

Funciones Principales

FunciónQué hace
_selectRole(role) 1. Llama AuthService.setRole() para escribir el rol en Firestore
2. Marca hasCompletedOnboarding: true en el documento del usuario
3. Navega a AuthGate limpiando el stack completo (pushAndRemoveUntil)
Por qué limpiar el stack: RoleSelectionScreen está encima de AuthGate en el stack, así que AuthGate no se "ve" aunque se reconstruya debajo.

Componente _RoleCard

Tarjeta de selección animada con:

  • AnimatedOpacity: la tarjeta no activa se vuelve semitransparente (45%) mientras la otra está cargando
  • CircularProgressIndicator en el ícono mientras se guarda
  • InkWell deshabilitado durante la carga (onTap = null)

Propósito

Permite a un usuario vincularse a un tutor mediante un código alfanumérico de 6 caracteres generado desde el panel del tutor. La vinculación habilita la supervisión en tiempo real.

FunciónQué hace

7. Servicios Core

7.1 AuthService

Archivo: lib/core/services/auth_service.dart

Servicio estático que centraliza toda la lógica de autenticación y vinculación tutor ↔ usuario.

Sistema de Roles (3 capas de fallback)

  1. Lee el campo role del documento Firestore
  2. Si vacío: infiere el rol por la presencia de campos estructurales (linkedPatients → tutor, sino → usuario)
  3. Si es un string legacy (usuario_tea, tdah, etc.): migra automáticamente a 'usuario'
La migración on-the-fly se escribe en Firestore para que no se repita en futuros accesos, sin necesitar scripts de migración batch.

Sistema de Códigos de Invitación

FunciónQué hace
generateInvitationCode() Genera código de 6 chars y lo guarda en la colección 'invitationCodes' con TTL de 7 días. El alfabeto excluye 0/O/1/I/l para facilitar la transcripción manual.
validateInvitationCode(code) Verifica: existencia, status == 'active', expiresAt en el futuro. No consume el código. Retorna {valid, tutorName, tutorId} o {valid: false, reason}.
acceptInvitationCode(code) Batch atómico: marca código 'used' + crea linkedTutors/{tutorId} en el doc del usuario + guarda acceptedInvitationCode. Segunda verificación de race condition antes del batch.
removePatientLink(patientId) Soft-deactivation: marca linkedTutors/{tutorId} como 'inactive' sin borrar el documento.

7.2 PomodoroService

Archivo: lib/features/tda_focus/services/pomodoro_service.dart

ChangeNotifier que gestiona el timer Pomodoro con persistencia y notificaciones.

Diseño del Temporizador

En lugar de un contador decremental, persiste _endTime (DateTime de finalización) y calcula el tiempo restante como _endTime - DateTime.now() en cada tick (1 segundo). Esto garantiza precisión aunque el hilo de UI se congele brevemente.

Persistencia (SharedPreferences)

ClaveTipoContenido
pomodoro_end_time_msintTimestamp epoch de finalización (solo cuando running)
pomodoro_total_duration_msintDuración total seleccionada en ms
pomodoro_remaining_msintTiempo restante en ms (usado al restaurar paused)
pomodoro_statusintÍndice del enum PomodoroStatus

Restauración al Abrir la App (_restoreState)

CasoComportamiento
running + endTime en el pasadoEl timer terminó en background → status = finished, muestra notificación
running + endTime en el futuroReanuda el ticker desde el tiempo real correcto
pausedRestaura remaining sin endTime
finishedMantiene el estado para que la UI lo muestre

Detección de Volver al Primer Plano

Implementa WidgetsBindingObserver. Al detectar AppLifecycleState.resumed, llama _refreshFromPersistence() para corregir cualquier drift acumulado en background.

7.4 StreakService — Racha Diaria

Archivo: lib/features/tda_focus/services/streak_service.dart

Reglas de la Racha

Diferencia de días (today - lastStreakDate)Resultado
== 0 (misma fecha)Mantiene la racha actual (ya completó una tarea hoy)
== 1 (día siguiente)Incrementa en 1 (racha consecutiva)

Usa _stripTime() para normalizar las fechas a medianoche antes de comparar, ignorando la hora exacta del día.

Implementado con FirebaseFirestore.runTransaction() para atomicidad en caso de escrituras concurrentes.

- `lib/main.dart` вЂ” Initializes Firebase, notifications, PomodoroService, then mounts `AuthGate`

## Firebase Structure

### Firestore Rules (CRITICAL)
- `firestore.rules` вЂ” Role-based access control
- Tutor can read/write linked patient data via `linkedTutors/{tutorId}` subcollection
- Patient owns their data; tutor has supervised access
- `invitationCodes` collection for tutor-patient linking

### Collections
```

  /tasks/{taskId}
  /progress/{progressId}
  /communications/{commId}

  /activityLog/{logId}
  /pictogramSettings/{settingId}
  /linkedPatients/{patientId}

  /invitationCodes/{codeId}

/sensitiveData/{patientId}  в†ђ Tutor-only clinical data
/invitationCodes/{code}     в†ђ Global invitation codes
/pictogramTemplates/{templateId}  в†ђ Public pictogram bank
/notificationQueue          в†ђ Collection group for scheduled notifications
```

## Key Conventions

### Role-Based Routing
- **Never** bypass `AuthGate` or `RoleDispatcher`
- Role is stored in Firestore `users/{uid}.role` ('tutor' or 'usuario')
- `RoleDispatcher` uses widget replacement (not Navigator.push) to avoid web history issues

### Pictogram Settings
- `pictogramSettings` subcollection stores per-user overrides (visibility, category)
- Special doc `_features` stores feature flags (`featurePictogramas`, `featureFoco`)
- Tutor can modify these for linked patients

### Activity Log
- Write-only for patient, read-only for tutor
- Never delete activity logs (audit trail)

### Soft Delete for Tasks
- Tasks use `deletedByUser: true` flag instead of hard delete
- Tutor retains history even after patient "deletes"

## Common Gotchas

1. **SHA-1 mismatch** вЂ” Google Sign-In fails if SHA-1 in Firebase Console doesn't match your keystore
2. **IndexedStack + ValueKey** вЂ” Tutor dashboard uses this pattern to force rebuild when switching patients
3. **Cloud Functions secrets** вЂ” `GEMINI_API_KEY` is a Firebase secret, not env var in production
4. **Notification Queue** вЂ” Uses collection group query on `notificationQueue` with composite index (see `firestore.indexes.json`)
